Python核心笔记day1--面向对象

发表于 2016-10-23 | 分类于 Python | | 阅读次数

创建一个类：

class MyNewObject(bases):

    'write something'

    class_suite

所有新式类必须至少继承一个父类，参数bases可以是一个或多个父类；

object是所有类之母，没有声明则默认作为新类的父类；

变量进行实例化时，赋值在语法上不是必须的，但如果你没有把这个实例保存到一个变量中，它就没用了，会被自动垃圾收集器回收

类的self 参数，它在所有的方法声明中都存在，这个参数代表实例对象本身，当你用实例调用方法时，由解释器悄悄地传递给方法的，它是自动传入的

init实例化时被调用，可以认为实例化是对init()的一种的调用

子类和基类构造器的问题：若子类重写基类的构造器，基类的构造器就不会被自动调用了，然而还需要自己的构造器，则需要在子类的构造器中显示调用基类的构造器：
class EmplAddrBookEntry(AddrBookEntry):

'Employee Address Book Entry class'

def __init__(self, nm, ph, id, em):

    AddrBookEntry.__init__(self, nm, ph)

    self.empid = id

    self.email = em

常用术语

1. 抽象/实现

抽象指对现实世界问题和实体的本质表现，行为和特征建模，建立一个相关的子集，可以用于描绘程序结构，从而实现这种模型

2. 封装/接口

合成扩充了对类的描述，使得多个不同的类合成为一个大的类，来解决现实问题

3. 派生/继承/继承结构

  • 派生描述了子类的创建，新类保留已存类类型中所有需要的数据和行为，但允许修改或者其它的自定义操作，都不会修改原类的定义。

  • 继承描述了子类属性从祖先类继承这样一种方式。

  • 继承结构表示多“代”派生，可以描述成一个“族谱”，连续的子类，与祖先类都有关系。

    4. 泛化/特化

  • 泛化表示所有子类与其父类及祖先类有一样的特点
  • 特化描述所有子类的自定义，也就是，什么属性让它与其祖先类不同。

    5. 自省/反射

    自省表示给予你，程序员，某种能力来进行像“手工类型检查” 的工作，它也被称为反射。这个性质展示了某对象是如何在运行期取得自身信息的。

类

类的数据属性，即静态变量，不依赖于任何实例，

Python严格要求，有实例，方法是不能被调用的，方法必须绑定（到一个实例）才能直接被调用

查看类的属性

  1. dir(类名)

  2. 类.dict

一个类的常见属性：

  • C.name 类Ｃ的名字（字符串）

  • C.doc 类Ｃ的文档字符串

  • C.bases 类Ｃ的所有父类构成的元组

  • C.dict 类Ｃ的属性

  • C.module 类Ｃ定义所在的模块（1.5 版本新增）

  • C.class 实例Ｃ对应的类（仅新式类中）

文档字符串不能被派生类继承，就是说派生类必须有自己的文档字符串

访问一个类属性的时候，Python 解释器将会搜索字典以得到需要的属性。 如果在dict中没有找到， 将会在基类的字典中进行搜索，采用“深度优先搜索”顺序。基类集的搜索是按顺序的， 从左到右，按其在类定义时，定义父类参数时的顺序。

调用type(instance)查看实例的类

实例属性

能够在“运行时”创建实例属性，是Python类的优秀特性之一，Python不仅是动态类型，而且在运行时，允许这些对象属性的动态创建，但是如果该条件语句块并未被执行，属性也就不存在，而你在后面的代码中试着去访问这些属性，就会有错误发生

构造器是最早可以设置实例属性的地方，再没有比这更早的可以设置实例属性的机会了

如果定义了构造器，它不应当返回任何对象，因为实例对象是自动在实例化调用后返回的。相应地，init()就不应当返回任何对象（应当为 None）；

实例仅有两个特殊属性,对于任意对象 I:

  • I.class 实例化 I 的类

  • I.dict I 的属性
